Problem

BPO team has no structured way to categorize invalid COAs. All invalid COAs look the same — no differentiation between "not a COA at all" vs "insufficient data" vs "suspicious." No distinction between system-detected flags and human-confirmed issues.

Issue Category Values (BR-1)
Not a COA Out-of-Scope Insufficient Data Suspicious COA Lab Error Duplicate Unreadable
Business Rules
BR-1Issue Category values as listed above.
BR-2System-flagged (auto-detected) shows "System Flagged" badge + inline action bar. Human-confirmed shows "Validation Issue" badge + category.
BR-3COA with confirmed Issue Category receives new status "Validation Issue" — distinct from Violative, Expired, Missing, Compliant.
BR-4Issue Category selectable from three-dot actions menu ("Flag Issue") and from COA Details modal.
BR-5Issue Category visible as tooltip on COA Status badge showing reviewer + timestamp.
BR-6"Validation Issue" added to Compliance Issue filter dropdown.
Acceptance Criteria
☐Issue Category field exists with all 7 values
☐"Validation Issue" COA status exists, distinct from all others
☐BPO can set Issue Category from three-dot menu and COA Details modal
☐COA Status badge tooltip shows category, reviewer, timestamp
☐System-flagged COAs show distinct "System Flagged" badge with Confirm/Dismiss actions
☐"Validation Issue" filterable in Compliance Issue filter
☐Validation Issue COAs excluded from compliant counts
