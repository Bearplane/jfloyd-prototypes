Designer notes — Quick-add as a power-user shortcut

Command-palette pattern

Familiar shape: ⌘K opens, type to filter, ↩ to add. Linear / GitHub / Notion users will recognize it instantly.

Triggered from the blank canvas in 3 ways: clicking the inline smart-add bar, ⌘K anywhere in the case, or the "Quick-add for power users" link in the empty zone.

Grouped, scoped, ranked

Results are grouped by source (same vocabulary as the targeting picker) so the analyst's mental model carries across both flows.

Default scope is this merchant (badge: "Black Bear Wellness"). Click the scope pill to widen — useful when the analyst remembers a SKU but not which merchant.

The "Past audit history" section quietly surfaces resolved cases — useful pattern detection ("this is the third time this gummy line failed") without forcing the analyst to leave.

Stay-and-add UX

⇧↩ = add and keep modal open — for the analyst stamping out 5 items in a row. The footer count goes up, the input is cleared, focus stays.

Plain ↩ adds and closes for the one-shot case. The "Add another" button is for users who don't know the keyboard shortcut.

Resolution chip shows in each row so adding is never a leap of faith — the analyst sees the default action that will be requested of the merchant before clicking.
