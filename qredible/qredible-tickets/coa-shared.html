❌ Today: upload 5 times

Creates 5 separate COA records. If the batch expires or is updated, each copy must be updated individually. Copies drift out of sync. No way to see which products share a batch.

✅ After QP-3546: one record, shared

Upload COA once to Remedy Shake (the source). Set the other 4 products to Assignment Level = Shared, pointing at Remedy Shake. One record. All reflect source status in real time. Clicking any dashed badge navigates straight to the source COA.

1 — Shared vs Inherited: What's the Difference?

Assignment Level Scope Typical Use "From" selector COA record
Inherited Sibling variants under the same parent only Weight variants of the same strain (7g, 28g inherit from 3.5g) Siblings only (same parent) 1 record, referenced by siblings
Shared New Any product in the full catalog Same batch across different product forms (flower → pre-rolls → shake → smalls) Full catalog search by name, SKU, or classification 1 record on source, all others reference it — no copies
Both use a dashed badge in the compliance table. The tooltip and "From" navigation differ — Inherited shows the sibling variant name; Shared shows the source product name with a clickable link.
