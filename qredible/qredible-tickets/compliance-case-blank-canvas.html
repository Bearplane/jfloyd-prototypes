Designer notes — Why empty looks like this

Encouraging, not intimidating

The empty zone uses a subtle violet→blue gradient and an avatar-sized "+" rather than a generic dashed rectangle. The headline counts the merchant's flagged items — this is the cue that "you have stuff to find here, it's not a blank page."

Quick-add cards mirror the source categories the targeting picker will use, so the analyst's mental model carries forward.

Three ways forward

Browse — click a quick-add card → opens targeting picker scoped to that source.

Inline — paste a SKU/finding ID into the smart-add bar, hit ↩ to add (no full picker).

Manual — small link for one-off items not yet captured by the system.

Defaults that matter

Priority defaults to Medium, SLA auto-fills to 7 days. High = 3d, Low = 14d (matches existing convention).

Title is optional in the empty state — auto-generated on send if left blank ("[Merchant] — [Date]").

Send button stays disabled until ≥ 1 item; Save Draft works at zero items so the analyst can come back later.
