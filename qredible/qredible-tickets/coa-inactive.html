Problem

When a merchant uploads a replacement COA, the old expired COA remains visible and counted in compliance status. This creates noise — reviewers are distracted by historical COAs that are already superseded, and compliance looks worse than it actually is. There is no default "optimized view" showing only COAs that need attention.

Business Rules
BR-1A COA is automatically marked Inactive when: (a) expiration date has passed AND (b) another non-expired COA is assigned to the same product.
BR-2Expired COA with no replacement on file is NOT Inactive — remains Expired (still needs attention).
BR-3Inactive COAs are excluded from compliance status counts, default export, and Optimized View.
BR-4Optimized View is the default state. Must render as a view toggle — NOT a filter chip. Avoid making users feel they are missing data.
BR-5Users can toggle to "All COAs" to see inactive ones (dimmed row treatment, cannot be selected).
BR-6"Inactive" added to Compliance Issue filter dropdown for explicit include/exclude.
BR-7COA count badge on Products tab reflects only active COA count in Optimized View.
Acceptance Criteria
☐COAs meeting BR-1 criteria are automatically marked Inactive by nightly job
☐Expired COAs with no replacement remain Expired (not Inactive)
☐Inactive COAs excluded from compliance status counts
☐Optimized View is default; renders as view toggle not filter chip
☐Toggling "All COAs" reveals inactive rows with dashed badge, dimmed treatment
☐"Inactive" option in Compliance Issue filter
☐Exports exclude inactive COAs by default in Optimized View
☐COA count badge on Products tab reflects only active COAs in Optimized View
☐Footer row count shows "X active COAs · Y inactive hidden" in Optimized View
