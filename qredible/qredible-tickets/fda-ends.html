Problem

FDA requires PMTA authorization for all ENDS products. Westtown's sponsor bank (via Jason Hatch) needs Qredible to surface whether merchant vape products are on the FDA authorized list. Currently no mechanism exists.

Authorization Status Values
✓ FDA Authorized Match ≥85%
✕ Not on FDA List No match ≥85% → auto-case
⚬ Unknown ENDS class, insufficient name data
Business Rules
BR-1FDA list maintained manually. Seeded from Brian's Mar 18 FDA PDF attachments. API automation is Phase 10+.
BR-2Match via Levenshtein distance, normalized names. ≥85% = positive match.
BR-3Only tobacco/vape classified products are evaluated. Traditional Tobacco = N/A (no ENDS check).
BR-4"Not on FDA List" auto-creates compliance case: Critical severity, Block enforcement, 0-day grace period.
BR-5Check triggers: (a) on classification to tobacco/vape, (b) on FDA list update, (c) manual from three-dot menu.
BR-6Match confidence stored on product record with matched FDA record ID.
