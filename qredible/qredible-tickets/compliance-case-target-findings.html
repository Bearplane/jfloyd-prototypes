Designer notes — How targeting works

Sources are the spine

The left rail organizes findings by source category rather than by severity or date. This matches the analyst's mental model: "I want to handle the violative COAs first, then the Q-ScanAI stuff."

Each rail item shows the scoped count (filtered to the case's merchant) and a selection badge when items from that source are picked. Switching sources preserves selections across the picker.

Resolution previewed up front

Each row shows the default resolution the system would apply if added — a resolution-chip. The analyst sees "Replace COA + reformulate" before adding, instead of after.

Defaults derive from itemType per QP-3593: COA_VIOLATIVE → Replace COA, COA_EXPIRED → Upload replacement, PRODUCT_MISSING_COA → Upload to MyCOA, COMPLIANCE_FINDING → Provide corrected URL. Editable per-item after add (see drawer in mockup 2).

Multi-source workflow

Selections accrue across sources. The footer is always visible with a live count, source-tagged chips, and the green "Add N" CTA. No "Apply" trap — you can switch sources, refine filters, and the selection persists.

The footer count syncs back to the blank canvas's Case Items (N) on close.
