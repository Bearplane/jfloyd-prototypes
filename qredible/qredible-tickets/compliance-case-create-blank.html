Where does + New Case live?

Today, every compliance case is born from items: an analyst selects products / findings, then clicks "Add to case." This v1 cut adds the inverse — start from a blank case for one merchant, then jump back into the existing tabs to attach items. The + New Case CTA needs to be visible anywhere a case might be conceived. This page maps the three v1 entry points.

⏸
No multi-merchant cases in v1
Each compliance case targets one merchant. The bulk action bar's "Create Case (N merchants)" pattern from QP-3583 Phase 2 isn't part of this scope — single-merchant only.
If a sweep is needed: the analyst opens the Cases tab, files three single-merchant cases (one per merchant) using entry points 1–3. v2 may revisit batched creation pending real-world feedback.
Why deferred
Multi-merchant cases complicate SLA, resolution, and merchant-side surfacing in ways that don't pay off until we see real usage. v1 ships the simpler primitive; v2 considers batched creation if it's worth it.
