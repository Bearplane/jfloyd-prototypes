Evolution Roadmap

Three phases from MVP to full enterprise compliance platform.

Phase 1 — MVP
QP-3579 to QP-3586 · Target: April 2026
✅ Data model (PlComplianceCase, CaseItem, CaseResponse)
✅ Case CRUD API (10 endpoints)
✅ Create Case from Q-ScanAI findings
✅ Create Case from Product Compliance table
✅ Active Cases column in We Monitor
✅ Internal case management screen (/qmonitor/cases)
✅ Merchant invite flow
✅ Merchant compliance portal (/comply)
Phase 2 — Enhanced
Post-MVP · Q3 2026
○ SLA notifications (email + in-app approaching/breached)
○ Case audit trail (full activity log)
○ Dispute resolution workflow (state machine)
○ Case templates per violation type
○ Bulk processor actions (reassign, close, SLA)
○ Reporting dashboard (cases by merchant, SLA breach trends)
Phase 3 — Advanced
Future · Roadmap TBD
○ Sanctions screening → case items (QP-3469 Vital 4)
○ Reputation events → case items (BBB, CourtListener)
○ Merchant-assisted COA matching via portal
○ Custom case templates per processor
○ Mobile app support (PWA or native)
○ API access for enterprise processors
