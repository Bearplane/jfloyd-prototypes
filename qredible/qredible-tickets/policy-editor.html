Why this is one edit instead of three tickets — Pre-pivot, adding a new checklist item meant (a) a new attribute row in the Regulatory Attributes table, (b) a rule definition in the Compliance Rules Engine, and (c) a code change to map it into the scanner. Now it is one line in the policy doc, versioned by the editor, and every downstream system (scanner, onboarding, MRI, reports) reads it from the same source.
