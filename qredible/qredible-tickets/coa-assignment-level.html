QP-3544 — COA Assignment Level · Mockup v3

1 — COA Assignment Level Values (set in Product Builder only — not a table column)

Individual default
This product requires its own COA assigned directly to it. Used for standalone products and for variant rows that each need their own COA (e.g. different strains). COA Status badge = solid outline.
Shows: COA Validated or Missing
Not Required
This product does not require a COA at all (e.g. accessories, apparel). No COA compliance is tracked. COA Status shows N/A — excluded from all COA counts.
Shows: N/A in COA Status
Child set on parent row
Set on a parent/container row. Children each need their own COAs. Parent shows an aggregated status rolled up from all Individual children. COA Status badge = dashed outline (rolled up, not directly assigned).
Parent shows: COA Validated ⤵ or Missing ⤵
Inherited set on variant row
This variant shares a COA with another product. An "Inherits COA From" selector appears to choose the source (sibling variants only). Does not count toward Missing totals. COA Status badge = dashed outline.
Shows: COA Validated ⤵ or Missing ⤵
Bundle
Product composed of other catalog products. No COA assigned directly — status derived from component products. Does not count toward Missing totals. COA Status badge = dashed outline.
Shows: COA Validated ⤵

Inheritance direction — two valid patterns

Option A — Children aggregate UP

Parent → Child
Strain A 3.5g → Individual ← COA here
Strain A 7g → Inherited from Strain A 3.5g
Strain B 3.5g → Individual ← COA here
Parent status = rollup from all Individual children

Option B — COA flows DOWN from parent

Parent → Individual ← COA lives here
Variant A → Inherited from Parent
Variant B → Inherited from Parent
Variant C → Inherited from Parent
Parent owns the COA; variants reflect its status

COA Assignment Level is not a column in the compliance table. It is only set and visible inside the Product Builder modal. The dashed badge style in the table is the only visual indicator of inherited/rolled-up COA status.
