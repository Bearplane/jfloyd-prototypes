1 — COA Assignment Level — What Each Value Means

Individual — default for all products
This product requires its own COA directly assigned to it. Used for:
• Standalone products with no children
• Variant rows that each need their own COA (e.g. different strains)

COA badge = solid outline (directly assigned).
Shows: COA Validated or Missing
Child — set on the parent row
Set on a parent/container row to say: "I'm just a grouping row — my children each need their own COAs. Roll their statuses up to me."

The parent row shows an aggregated (rolled-up) status derived from all Individual children. Dashed badge = derived, not directly assigned.
Parent shows: COA Validated ⤵ or Missing ⤵ (dashed = rolled up from children)
Inherited — set on a variant row
Set on a variant row that doesn't need its own COA — it shares a COA with another product. You select which product it inherits from via a "Inherits from" dropdown that appears when this is selected.

Typically used for weight variants of the same strain (7g, 28g inherit from the 3.5g). Does not count toward Missing totals.
Shows: COA Validated ⤵ or Missing ⤵ (reflects source product's status)
Bundle — set on bundle products
Set on a product that is composed of other existing catalog products (e.g. a gift set containing a vape + tincture + pre-roll). No COA needed directly — status derived from the component products.

Does not count toward Missing totals.
Shows: COA Validated ⤵ (all components validated)

How inheritance direction works

Option A — Children aggregate UP

Parent → Child
Child 1 (Strain A 3.5g) → Individual ← COA here
Child 2 (Strain A 7g) → Inherited from Child 1
Child 3 (Strain B 3.5g) → Individual ← COA here

Parent status = aggregate of all Individual children

Option B — Parent COA inherited DOWN by variants

Parent → Individual ← COA lives here
Child 1 (Variant A) → Inherited from Parent
Child 2 (Variant B) → Inherited from Parent
Child 3 (Variant C) → Inherited from Parent

Parent owns the COA; all children point at parent

Key distinction: "Child" (set on parent) = children own COAs, parent aggregates UP. "Individual on parent + Inherited on children" = parent owns COA, status flows DOWN. Both are valid patterns — the assignment level on each row determines which direction.
