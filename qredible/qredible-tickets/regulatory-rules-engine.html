Unified Compliance Ontology: Regulatory attributes are the controlled vocabulary. One attribute can bind to multiple data elements (a classification, a COA analyte, a website finding type). One rule can reference multiple attributes. When a product matches any binding for an attribute, it inherits that attribute — and every rule referencing that attribute will evaluate it. The Product Compliance Status column aggregates all applicable rule results and shows the most severe outcome. A product needs to pass every applicable rule to show ✓ Compliant.

Attributes are the building blocks — they don't DO anything by themselves. They're tags. The rules engine evaluates them. The Bound To column shows every data element that carries this attribute. When a product matches any binding condition (e.g., is classified as "Disposable Vape", or a COA shows Mitragynine detected), it inherits the attribute. The rules engine then evaluates all rules referencing that attribute and aggregates the results into the Product Compliance Status column.

Rules are the evaluation logic. They reference regulatory attributes and external data sources (like the FDA ENDS list). The visual rule builder uses IF / AND / THEN blocks. Rules run automatically when: a product is classified, a COA is uploaded, a website scan completes, or manually triggered.

The "Applies to" count shows real-time scope — how many products across how many merchants are currently affected by this rule. When a new merchant onboards and has vape products classified as "Disposable Vape", they are immediately included in the next rule evaluation run. The regulatory team has full visibility into who is affected before changing any rule.

This is the end result. The Product Compliance Status column aggregates all applicable rule evaluations into a single, scannable signal per product. Key takeaways from this table:

🏛️ FDA Authorization rows: JUUL Virginia Tobacco Pod shows FDA Authorized (it's on the list). Generic Disposable Vape shows PROHIBITED (not on the list) with a tooltip showing the specific rule and failure reason.

⚖️ Kratom row demonstrates the critical insight: Kratom Extract 50x has a valid COA (Attached) and an Approved lab disposition — the lab analysis is clean. But it is still PROHIBITED at the product level because state law bans it. COA compliance and legal compliance are orthogonal concerns. The tooltip explicitly surfaces this distinction so compliance reviewers understand why a "Approved" product is flagged.

Hovering the compliance status cell (shown as visible callouts here) displays which specific rules contributed to the status, with categorical icons: 🏛️ for government authorization checks, ⚖️ for legal/ban checks, 🧪 for lab-based checks. The Override action allows authorized users to manually exempt a product with a documented reason.
